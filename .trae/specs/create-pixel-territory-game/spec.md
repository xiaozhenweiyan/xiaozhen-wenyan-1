# 像素领地争夺战 - 网页游戏 Spec

## Why
用户想要一个像素风格的多人网页游戏，可以创建/加入房间，在2D方块地图上创建领地，分享给朋友一起玩。需要部署到GitHub Pages作为静态网页。

## What Changes
- 创建一个完整的像素风格网页游戏，包含：开始界面、临时账户注册、房间创建/加入、房间设置、2D方块地图、领地创建系统
- 使用纯前端技术（HTML/CSS/JavaScript），无后端服务器
- 使用 WebSocket 或 PeerJS 等 P2P 技术实现多人联机
- 部署到 GitHub Pages 静态网页托管

## Impact
- 新建项目：完整的前端游戏项目
- 技术栈：HTML5 Canvas + CSS 像素风格 + PeerJS (WebRTC P2P)
- 部署：GitHub Pages 静态托管

## ADDED Requirements

### Requirement: 开始界面
系统 SHALL 提供像素风格的开始界面，包含游戏标题和"开始游戏"按钮。

#### Scenario: 用户打开游戏
- **WHEN** 用户访问游戏网页
- **THEN** 显示像素风格的开始界面，包含游戏标题"像素领地争夺战"和"开始游戏"按钮

### Requirement: 临时账户注册
系统 SHALL 提供临时账户注册功能，用户输入昵称即可注册，无需密码和邮箱。

#### Scenario: 用户注册临时账户
- **WHEN** 用户点击"开始游戏"后进入注册界面
- **THEN** 显示昵称输入框和"确认"按钮，输入昵称后即可注册临时账户并进入主菜单

#### Scenario: 昵称为空
- **WHEN** 用户未输入昵称就点击确认
- **THEN** 显示提示"请输入昵称"

### Requirement: 主菜单
系统 SHALL 提供主菜单界面，包含"创建房间"和"加入房间"两个选项。

#### Scenario: 注册成功后进入主菜单
- **WHEN** 用户成功注册临时账户
- **THEN** 显示主菜单，包含"创建房间"和"加入房间"按钮

### Requirement: 创建房间与房间设置
系统 SHALL 提供房间创建功能，包含房间设置界面。

#### Scenario: 用户点击创建房间
- **WHEN** 用户在主菜单点击"创建房间"
- **THEN** 进入房间设置界面

#### Scenario: 房间设置界面
- **WHEN** 用户进入房间设置界面
- **THEN** 显示以下设置项：
  - 房间名字（必填）
  - 房间类型：公有/私有（默认公有）
  - 密码（仅选择私有时显示，可选）
  - 邀请码（自动生成，格式为XXXXX-XX，大写英文和数字随机组合，不可修改）
  - "确认"按钮

#### Scenario: 房间名字为空
- **WHEN** 用户未填写房间名字就点击确认
- **THEN** 显示提示"请输入房间名字"

#### Scenario: 邀请码格式
- **WHEN** 创建房间时生成邀请码
- **THEN** 邀请码格式为 XXXXX-XX，其中X为大写英文或数字的随机组合

### Requirement: 2D方块地图
系统 SHALL 提供以草地为底、四周被海水环绕的2D方块地图。

#### Scenario: 进入游戏地图
- **WHEN** 用户确认房间设置后
- **THEN** 显示2D方块地图，中间区域为草地，四周为海水，所有元素以方块为单位

### Requirement: 领地创建
系统 SHALL 允许玩家在地图上点击方块来创建领地。

#### Scenario: 创建领地
- **WHEN** 玩家点击地图上的草地方块
- **THEN** 该方块变为该玩家颜色的半透明方块，表示该领地属于该玩家

#### Scenario: 已有领地方块
- **WHEN** 玩家点击已被其他玩家占据的方块
- **THEN** 不执行任何操作，该方块保持原拥有者的颜色

### Requirement: 加入房间 - 公共房间列表
系统 SHALL 提供公共房间列表供玩家浏览和加入。

#### Scenario: 进入加入房间界面
- **WHEN** 用户在主菜单点击"加入房间"
- **THEN** 显示公共房间列表，左上角有"返回"按钮，右上角有"加入私有房间"按钮

#### Scenario: 加入公共房间
- **WHEN** 用户点击列表中的某个公共房间
- **THEN** 直接进入该房间的游戏地图

### Requirement: 加入房间 - 私有房间
系统 SHALL 提供私有房间加入功能，通过邀请码和密码加入。

#### Scenario: 点击加入私有房间
- **WHEN** 用户点击右上角"加入私有房间"按钮
- **THEN** 显示邀请码输入框和密码输入框（可选），以及"加入"按钮

#### Scenario: 通过邀请码加入私有房间
- **WHEN** 用户输入正确的邀请码并点击加入
- **THEN** 进入该房间的游戏地图（若房间有密码则需同时输入正确密码）

#### Scenario: 邀请码错误
- **WHEN** 用户输入的邀请码不存在
- **THEN** 显示提示"房间不存在"

### Requirement: 开始游戏按钮
系统 SHALL 在房间人数大于等于2人时，仅在房主界面显示"开始游戏"按钮。

#### Scenario: 仅1人在房间
- **WHEN** 房间内仅有房主1人
- **THEN** 房主界面不显示"开始游戏"按钮

#### Scenario: 2人及以上在房间
- **WHEN** 房间内有2人或以上
- **THEN** 房主界面显示"开始游戏"按钮

### Requirement: 多人联机
系统 SHALL 使用 WebRTC P2P 技术实现多人联机，房主作为主机。

#### Scenario: 房主创建房间
- **WHEN** 房主确认房间设置
- **THEN** 房主作为 WebRTC 主机，等待其他玩家通过邀请码连接

#### Scenario: 其他玩家加入
- **WHEN** 其他玩家通过邀请码加入
- **THEN** 通过 WebRTC 连接到房主，实时同步地图状态

### Requirement: GitHub Pages 部署
系统 SHALL 将游戏部署到 GitHub Pages 作为静态网页。

#### Scenario: 部署完成后
- **WHEN** 用户访问 GitHub Pages URL
- **THEN** 可以正常游玩游戏

### Requirement: 像素风格美术
系统 SHALL 全局采用像素风格美术设计。

#### Scenario: UI和地图风格
- **WHEN** 用户浏览游戏界面
- **THEN** 所有UI元素、字体、地图方块均采用像素风格设计
